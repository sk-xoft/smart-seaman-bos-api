Authentication & Required Headers

Every /v1/** request must include the following headers:

HeaderExampleDescription
languageTHResponse language: TH or EN
devicemodelPostmanClient device model identifier
correlationidreq-trace-001Unique request trace ID for logging

Protected endpoints additionally require:

HeaderExampleDescription
AuthorizationBearer <jwt_token>JWT from POST /v1/login
JWT Flow: Call POST /v1/login → receive data.token → set Authorization: Bearer {token} on all protected requests.
Session is single-device: logging in elsewhere invalidates the previous token.

Response envelope: { "code": "...", "description": "...", "data": { ... } }
